Flux

通过Flux.range方法创建一个flux，来看下反应式规范中定义的几个角色以及相互交互

三种角色（规范中定义的是四种）：
Publisher、Subscriber、Subscription

Publisher：

请求Publisher开始下发数据，可以被调用多次，每次都会使用新的Subscription，每个Subscription只为一个Subscriber服务，一个Subscriber只能订阅一个Publisher
public void subscribe(Subscriber<? super T> s);

Subscriber: 

在Publisher#subscribe(Subscriber)调用之后被调用，直到Subscription#request(long)被调用才有元素流动，Publisher才会下发元素，其实也是通过Subscription这个中间者来通过request方法来下发元素，调用Subscriber的onNext方法下发
public void onSubscribe(Subscription s);
通过Subscription#request(long)时候调用来获取元素通知
public void onNext(T t);
当下发流程中出现错误一种失败终止状态，即使是再次调用Subscription#request(long)也不会触发任何事件
public void onError(Throwable t);
成功状态即使是再次调用Subscription#request(long) 也不会触发任何事件
public void onComplete();

Subscription：

Publisher不会发送任何的事件直到整个方法被调用
如果流结束，发布者可以发送的请求少于发送的请求，但是随后必须调用订阅者的onError或者onComplete
public void request(long n);
Publisher停止发送和清理资源的时候被调用，仍然可以发送数据以满足先前发出的信号需求
public void cancel();
